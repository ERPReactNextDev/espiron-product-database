# TDS Required Validation Fix

## Issue
Hindi required ang **TDS PDF** (View TDS link) bago mag-submit o mag-save ng SPF. Users can submit SPF without generating TDS first.

## Solution
Added validation sa both **Create SPF** at **Edit SPF** modes para i-require ang TDS PDF URL bago mag-submit.

## Changes Made

### 1. SPF Request Create (`components/spf-request-create.tsx`)

#### Added Validation in handleSubmit
```typescript
// Validate TDS PDF URL
if (!prod.__tdsPdfUrl || prod.__tdsPdfUrl.trim() === "") {
  toast.error(`Row ${i + 1}, Option ${j + 1}: TDS PDF is required. Please generate TDS first.`);
  return;
}
```

**Location:** Line ~714 (after TDS Brand validation)

#### Updated Button Disabled Condition
```typescript
disabled={
  isSubmitting ||
  (formData.item_description?.length || 0) === 0 ||
  formData.item_description?.some(
    (_, i) => !productOffers[i] || productOffers[i].length === 0,
  ) ||
  Object.values(productOffers).flat().some(
    (p: any) => !p.__priceValidity?.trim() || !p.__tdsBrand?.trim() ||
      !p.__tdsPdfUrl?.trim() ||  // ✅ Added TDS check
      (p.countries?.length > 1 && !p.__selectedBranch?.trim())
  )
}
```

## Validation Flow

### Before Submit/Save
1. ✅ Check if Price Validity is filled
2. ✅ Check if TDS Brand is selected
3. ✅ **Check if TDS PDF URL exists** (NEW)
4. ✅ Check if Branch is selected (for multi-country products)

### User Experience
- Submit/Save button is **disabled** kung walang TDS PDF
- Error toast message: `"Row X, Option Y: TDS PDF is required. Please generate TDS first."`
- User must click **"Generate TDS"** button para ma-enable ang Submit/Save
